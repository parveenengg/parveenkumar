Nov 2026 | Console-based Java Application

The Zoo Management System is a console-based Java application that simulates managing animals in a zoo environment. This project demonstrates object-oriented programming principles, file I/O operations, and serialization/deserialization of Java objects. This is a console application with no graphical user interface (GUI)—all interactions are menu-driven through the terminal.

Course Context: This project was completed as part of the "Amazon Junior Software Developer" course on Coursera. The repository represents completed coursework and is uploaded to GitHub to showcase learning and implementation.

PROJECT OVERVIEW

The application provides a comprehensive system for managing three types of zoo animals: Tiger (walks on land), Dolphin (swims in water), and Penguin (both walks and swims). Users can configure animal properties, simulate behaviors, and persist data using Java serialization.

FEATURES

  • Animal Management: Interact with three types of animals—Tiger, Dolphin, and Penguin—each with unique behaviors and properties
  • Property Configuration: Set and display specific properties for each animal type (stripes, speed, color, swimming status, etc.)
  • Behavioral Simulation: Display animal movements (walking/swimming) and eating behaviors
  • Data Persistence: Save and load animal states using Java serialization to maintain data between sessions

HOW THE MAIN MENU WORKS

The application uses a two-level menu system for intuitive navigation:

Main Menu (Zoo Animal Choice Menu):

  • Tiger — Select to work with tiger properties and behaviors
  • Dolphin — Select to work with dolphin properties and behaviors
  • Penguin — Select to work with penguin properties and behaviors
  • Save animals to file — Saves the current state of all three animals to disk using serialization
  • Display saved animals from file — Loads and displays previously saved animal data from files

Animal Details Menu: After selecting an animal, you'll see a sub-menu with four options:

  • Set properties — Configure animal-specific attributes (e.g., stripes for Tiger, color for Dolphin, speeds for Penguin)
  • Display properties — View all current animal characteristics including age, height, weight, and animal-specific properties
  • Display movement — Show animal movement behavior (walking for Tiger, swimming for Dolphin, walking or swimming for Penguin based on its state)
  • Display eating — Show animal eating behavior and completion message

CONCEPTS COVERED

Object-Oriented Programming (OOP):

  • Inheritance: Tiger, Dolphin, and Penguin extend the abstract Animal class
  • Abstraction: Abstract Animal class defines common properties and behaviors
  • Interfaces: Eat, Walk, and Swim interfaces define behavioral contracts
  • Encapsulation: Private fields with public getter/setter methods
  • Polymorphism: Interface implementations and method overriding

File I/O: Reading from and writing to files, creating and reading .txt files for data storage

Serialization and Deserialization: Using ObjectOutputStream and ObjectInputStream to write and read objects from files, implementing the Serializable interface

Exception Handling: Try-with-resources for automatic resource management, handling IOException and ClassNotFoundException

Control Flow: Menu-driven design using switch-case statements, do-while loops for menu navigation, and user input validation with Scanner
